# Mobile Parity Contract

CricketClubApp ships as three aligned surfaces:
- Website
- iOS app
- Android app

Parity rules:
- Every user-visible feature added to one surface must be reflected in the other two surfaces.
- Club-scoped data, scoring, archives, player profiles, availability, chat, and admin controls must stay consistent across all surfaces.
- Platform-specific UI may differ in layout, but the underlying workflows and data model must stay the same.
- Documentation updates should mention any platform-specific exception explicitly.
